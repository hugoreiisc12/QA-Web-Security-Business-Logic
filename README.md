# QA & Web Security – Análise de Lógica de Negócio

Projeto educacional focado na análise de lógica de negócio e controle de acesso
em um fluxo típico de e-commerce, sob a ótica de QA com mindset de segurança.

## Objetivo
Avaliar se regras críticas (planos, permissões e acesso a funcionalidades)
estão devidamente protegidas no backend ou se há confiança excessiva no client-side.

Essa análise está alinhada ao
OWASP Top 10 – Broken Access Control.

## O que este projeto NÃO é
- Não é exploração
- Não é bypass de segurança
- Não expõe sistemas reais
- Não concede acesso indevido

## O que este projeto demonstra
- Como QA pode identificar riscos de segurança
- A importância da validação server-side
- Análise de impacto técnico e de negócio
- Uso de automação como apoio à validação

## Estrutura
- Scripts demonstrativos e seguros
- Documentação técnica
- Fluxos de código e mindset
- Diagramas explicativos

## Aviso Ético
Todo o conteúdo deste repositório é estritamente educacional.
Nenhuma técnica aqui apresentada deve ser usada contra sistemas reais
sem autorização explícita.
