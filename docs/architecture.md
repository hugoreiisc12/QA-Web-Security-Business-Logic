# Arquitetura do Projeto 

Este projeto simula a análise de lógica de negócio em um fluxo de e-commerce,
com foco em controle de acesso e diferenciação de planos.

A análise considera três camadas:
- Frontend (camada não confiável)
- Backend (autoridade de regras)
- QA/Security (validação de risco)

O objetivo é avaliar se decisões críticas dependem indevidamente do client-side.
